# Security Policy

## Purpose of this repository

This repository exists as a public, documentation-first portfolio for Codestech product work. Its purpose is to present high-level business cases, product rationale, architecture direction, PMO logic and executive communication material for professional evaluation.

**"Este repositorio contem exclusivamente propriedade intelectual conceitual, diagramas de arquitetura em alto nivel e estudos de caso de negocios da Codestech. Nenhuma linha de codigo proprietario, chaves de API, segredos de producao ou dados reais protegidos pelo RGPD/LGPD de clientes estao ou serao expostos aqui."**

## What is allowed

- High-level architecture diagrams
- Strategic documentation and public-safe case studies
- Product positioning and business value framing
- Maturity descriptions, roadmaps and PMO-oriented planning
- Sanitized screenshots and media approved for public display
- Links to public portfolio material and GitHub Pages content
- Primary public portfolio URL: https://adrianoportuga.github.io/codestech-product-portfolio/

## What is not allowed

- Proprietary production source code
- Private repository content
- Credentials, tokens, secrets or keys
- Internal endpoints, internal URLs or unpublished environments
- Client-identifiable information
- Protected business data, operational logs or confidential assets
- Unsanitized screenshots, exports or recordings

## Data protection and governance

This portfolio follows a strict public-safe governance model:

- private production systems remain private
- sensitive implementation details are excluded by default
- client data is never published
- uncertain information must be marked as `to be confirmed`
- public documentation must privilege abstraction over exposure

The repository is intended to support professional evaluation without weakening technical, contractual or data-protection boundaries.

## Sensitive information policy

Sensitive information must never be committed, pasted or referenced in this repository, including:

- API keys
- production secrets
- internal system addresses
- real customer records
- confidential pricing or contract details
- private architecture details that materially increase exposure risk

If a detail is not clearly safe for public release, it should remain unpublished.

## Screenshots and media policy

- Screenshots must be sanitized before publication.
- Screenshots must not contain real names, email addresses, phone numbers, customer records, credentials or environment details.
- Diagrams and mockups should remain conceptual unless explicitly approved for publication.
- Media should support explanation, not expose operations.

## Reporting concerns

If you identify any content that appears sensitive, excessive or wrongly published, report it through a GitHub issue when appropriate or direct contact through a private channel so it can be reviewed and removed if necessary.

- Screenshots must be sanitized.
- No internal endpoints.
- No private repositories content.
- No real client data.
- No credentials or secrets.

## Public-safe operating principle

This repository demonstrates product, strategy and architecture thinking. Real production repositories, implementation assets and operational systems remain private by design.
